## 八、关键架构与约定（必读）

### 8.1 主题感知系统（AppClr）

项目的核心视觉系统，所有颜色**必须**通过主题感知方式获取，**禁止硬编码 Color 值**。

```
AppClr.of(context)               // 获取主题解析器
  .resolve(深色色, 浅色色)         // 罕见色用此方法
  .textPrimary                   // 常用色直接访问（深/浅自动切换）

context.appClr                   // 便捷扩展
```

- **两套色板**：`AppColors`（深色常量）+ `AppColorsLight`（浅色常量）
- **插值机制**：`ThemeAnimScope` 提供插值 `t`（0=深 1=浅），通过 `Color.lerp(dark, light, t)` 实现深浅渐变切换
- **GoldButton / DarkButton** 必须用 `c.resolve(深色, 浅色)`，不可硬编码

### 8.2 ConsumerStatefulWidget 动效组件模式

带动画的组件必须遵循此模式（参照 `gold_button.dart`）：

```dart
class MyWidget extends ConsumerStatefulWidget { ... }

class _MyWidgetState extends ConsumerState<MyWidget> {
  bool _animEnabled = true;  // 实例字段缓存

  @override
  Widget build(BuildContext context) {
    // build 中订阅全局配置（设置页切换开关时即时重建）
    _animEnabled = ref.watch(configProvider).valueOrNull?.animationsEnabled ?? true;
    // tap 回调读 _animEnabled 字段（避免在非 build 上下文调 ref.watch）
  }
}
```

### 8.3 构建产物命名规则（2026-07-22 起生效）

```
{程序名}_{版本号}_{状态}_{构建日期}_{构建序号}.{扩展名}
```

- 版本号在状态**之前**
- 示例：`Jeenith_2.10.4_fix_20260723_01.apk`
- GitHub Release 的旧版 assets 保留旧命名格式（不重新上传，保护已有下载链接）

### 8.4 Hard Constraints（不可违反的硬约束）

1. Release 构建必须使用专用 release keystore（非 debug 签名）
2. 所有卜算术必须使用 TrueRandom 引擎生成随机数
3. 桌面窗口尺寸：9:18.5 宽高比，clamp 在 380×600 ~ 1400×2789
4. GoldButton/DarkButton 必须用 `c.resolve()`，不可硬编码 Color
5. GoldButton 用单个 `ConstrainedBox(minWidth:88)`，DarkButton 用 `minWidth:72`，禁止 `maxWidth:double.infinity` + `SizedBox(width:double.infinity)`（会导致竖线坍塌，已复发 5 次）
6. Pinned SliverPersistentHeader 内禁止嵌套 SingleChildScrollView（会导致 viewport geometry 链断裂 + 空指针崩溃）
7. HistoryStore 操作必须用原子读-改-写模式（防快速连续卜算丢数据）
8. CustomPainter 必须显式 `dispose()` TextPainter（防 native handle 泄漏）
9. 版权行：`Copyright (c) 2026 Qore`（**禁止** "All rights reserved"，MIT 协议要求）

### 8.5 常见陷阱

- **GoldButton 竖线坍塌**：已复发 5 次（v2.3.1/v2.7.1/v2.10.0/v2.10.1/v2.10.3），根因是 tight(W<88) 矛盾约束。当前方案：LayoutBuilder 检测矛盾约束时降级渲染。**加 minWidth:88 防御时务必同时处理矛盾约束场景。**
- **CMake 增量构建不感知 .ico 变化**：Windows 图标更新后需 `flutter clean` 再构建
- **release_history.json commit 字段**：除 v1.1.2 外，所有版本在 GitHub Release 发布前留空，发布后回填

### 9.4 发布流程（标准步骤）

详见 [docs/工作流程.md §7.3](file:///d:/Code/Project/Qore/Jeenith/docs/工作流程.md)。摘要：

1. **构建产物**：APK（脚本自动）+ Windows zip（手动）归档到 `builds/`
2. **填写 release_notes**：新建 `builds/release_notes/release_notes_vX.Y.Z.md`
3. **更新 release_history.json**：新增条目（`isLatest:true, label:"Latest", status:"unpublished"`），旧版本降级为 `None/false`
4. **同步版本号引用**：CLAUDE.md / AGENTS.md / 工作流程.md → 新版本号
5. **Git commit + push**：commit message 格式 `feat: vX.Y.Z+<N> <摘要>`（多行 body 用多个 `-m`）
6. **GitHub Release**：`gh release create vX.Y.Z <apk> <zip> --title "..." --notes-file "..." --target main`
7. **回填 release_history.json**：补 url / publishedAt / downloadUrl / commit / `status:"published"`
8. **Git push 回填提交**

### 9.5 变更日志要求

每次修改后**必须**写 `log/update/{type}_{YYYYMMDD}_{NN}.md`（类型：feature/fix/refactor/config/docs/style/perf/test）。
