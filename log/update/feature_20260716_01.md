## 五、给协作者 Claude Code 的注意事项

### 1. 严禁恢复 "All rights reserved"
用户对版权清理非常敏感。**任何新建的 dart 文件**版权头只能是：
```dart
// Copyright (c) 2026 Qore
```
**不能**加 "All rights reserved."。Windows `Runner.rc` 的 `LegalCopyright` 同理。

### 2. 版本引用不硬编码
身份信息（组织 / 包名 / 版权 / 署名）以 `D:\Code\.Rules\OrganizationAndUser.md` 为准。修改前先读这份文件。

### 3. CustomPainter 必须显式 dispose TextPainter
项目硬性规则，防止 native handle 泄漏。新建 ritual 动画或 painter 时注意。

### 4. HistoryStore 原子读-改-写
快速连续卜算时容易丢数据，操作必须用原子模式。

### 5. 构建用脚本
```bash
cd mobile
flutter pub get && flutter analyze
pwsh -File scripts/build_apk.ps1 -Status release
```
**不要**手动 `flutter build apk` + 手动改版本号，会破坏版本前置策略。

### 6. Windows ZIP 需手动归档
`build_apk.ps1` 只处理 Android APK。Windows 桌面构建后需手动：
1. `Compress-Archive` 打包 `build\windows\x64\runner\Release\*`
2. 命名为 `Jeenith_<status>_<version>_<date>_<seq>_windows_x64.zip`
3. 放到 `builds/windows/`
4. 手动追加到两份 `build_history.json`

### 7. PowerShell 编码
所有 ps1 脚本含中文注释，必须用 `pwsh`（PowerShell 7+）执行，不能用 `powershell.exe`（5.x 默认 GBK 会报错）。

### 8. Commit Message 多行写法
在 PowerShell 中**不能**用 bash heredoc（`$(cat <<'EOF'...EOF)`），要用多个 `-m`：
```powershell
git commit -m "feat: vX.Y.Z+<N> 标题" -m "- 变更 1" -m "- 变更 2"
```

### 9. Release notes 格式
新建 `builds/release_notes/release_notes_vX.Y.Z.md` 时参考 v2.4.0 格式：概述 + 分章（背景/实施/影响文件/验证）+ 版本号更新 + 自上一版本提交表 + 下载表。

1. `CLAUDE.md` / `AGENTS.md` — 项目规则速览
