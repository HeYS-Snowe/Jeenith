# GoldButton 竖线坍塌第 5 次根治（v2.10.3）

**类型**：fix
**日期**：2026-07-22（最后修订 2026-07-22）
**影响范围**：mobile/lib/features/zhouyi/ui/zhouyi_page.dart、mobile/lib/shared/widgets/gold_button.dart、docs/频发BUG/GoldButton竖线坍塌.md

## 变更描述

v2.10.1 发布后用户报告周易卜算页面"起卦"按钮在小屏手机上仍坍塌为一条金线（第 5 次复发）。根因是 v2.10.1 的 `ConstrainedBox(minWidth:88)` 单层方案遗漏了 tight(W<88) 矛盾约束场景：周易 _buildActionBar 单行 4 按钮的总 minWidth（404px）在 320-411px 屏超过可用宽度（288-379px），Expanded 给 GoldButton 的 width<88 触发矛盾约束，ConstrainedBox.size 被 parent clamp 回 0 但 child 渲染 88 宽，造成 size=0/child=88 撕裂，release 模式渲染降级为竖线。

本次采用双层修复：
1. **布局层**：周易 _buildActionBar 弃用 `Expanded(GoldButton)`（矛盾约束的直接来源），改为「单行 + FittedBox(scaleDown) + 居中」——4 按钮按 intrinsic 宽度并排（`Row(mainAxisSize: min)`），外层 `Center + FittedBox(fit: scaleDown, alignment: center)`：大屏原尺寸居中留白、小屏整体等比缩小到一行容纳。同步 _PinHeaderDelegate extent 90→104→90
2. **防御层**：GoldButton 内部加 LayoutBuilder 检测 tight(maxWidth<88) 矛盾约束，矛盾时放弃 minWidth 强制按可用宽度渲染。仅矛盾时触发，其他 14 个调用点行为不变

## 方案演进

方案经过三次迭代（均在 2026-07-22 当天）：
1. **两行布局**（初版）：Row1=摇卦+重置，Row2=复制+分享，extent=104。用户反馈"按钮太宽"
2. **单行 + 横向滚动**：`SingleChildScrollView(horizontal) + ConstrainedBox + IntrinsicWidth + Row(center)`，extent=90。引发新崩溃——嵌套 viewport（`_RenderSingleChildViewport`）在 `pinned SliverPersistentHeader` 内导致 hit test 时 `sliver.geometry` 为 null，抛 `Null check operator used on a null value`（viewport.dart:886）
3. **单行 + FittedBox(scaleDown)**（最终）：`Center + FittedBox(fit: scaleDown) + Row(mainAxisSize: min)`，extent=90。无嵌套 viewport、无溢出、无空指针，大小屏均正常

## 修改详情

- [mobile/lib/features/zhouyi/ui/zhouyi_page.dart]：_buildActionBar 从单 Row + Expanded(GoldButton) → `Center + FittedBox(fit: scaleDown) + Row(mainAxisSize: min)` 单行 4 按钮；Container minHeight 90；_PinHeaderDelegate extent 90
- [mobile/lib/shared/widgets/gold_button.dart]：expandedBox 从 ConstrainedBox(minWidth:88) 单层 → LayoutBuilder 矛盾约束防御层；更新类注释（v2.10.1→v2.10.3，追加 tight(W<88) 矛盾约束场景说明）
- [docs/频发BUG/GoldButton竖线坍塌.md]：追加 v2.10.1 盲点章节 + v2.10.3 第五次根治章节（单行 + FittedBox 方案）+ 版本演进时间线 v2.10.3 行 + 预防措施 4-5 条（多按钮 ActionBar 布局原则：优先 FittedBox 缩放 + 禁止在 pinned SliverPersistentHeader 内嵌 SingleChildScrollView + 防御层不可移除）；更新顶部元数据

## 影响分析

- **周易页面**：4 按钮单行居中，大屏（≥480px）原尺寸左右留白，小屏（<480px）整体等比缩放到一行容纳，所有屏幕均不再触发矛盾约束、不崩溃
- **其他 14 个 GoldButton 调用点**：不受影响。LayoutBuilder 防御层仅在 tight(maxWidth<88) 矛盾时触发，其他场景（tight≥88 / loose / unbounded）走原 ConstrainedBox 分支
- **DarkButton**：未动，v2.10.2 AnimatedSwitcher 文字过渡动画完整保留
- **桌面端**：_buildActionBar 移动/桌面共用，≥600px 窗口单行居中布局视觉正常
- **_PinHeaderDelegate extent 一致性**：extent=90 == Container minHeight=90，无 paintExtent < layoutExtent 异常
