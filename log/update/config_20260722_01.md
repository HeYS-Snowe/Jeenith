# 构建产物命名规则调整：状态与版本号位置互换

**类型**：config
**日期**：2026-07-22 11:30
**影响范围**：mobile/scripts/build_apk.ps1、mobile/scripts/archive_history.py、builds/build_history.json、mobile/build_history.json、docs/FLUTTER_APK_BUILD_PIPELINE.md、docs/工作流程.md、docs/项目文档/05-部署上线 Deployment/部署运维手册 Deployment Guide.md、全局个人规则 C:\Users\aaa\.trae-cn\user_rules.md

## 变更描述

将构建产物命名规则中的「构建状态」与「版本号」字段位置互换，让版本号紧跟程序名后，状态字段后置。新格式更符合"先识别版本、再识别状态"的浏览直觉。

### 命名规则变化

| 项目 | 旧格式 | 新格式 |
|------|--------|--------|
| 中文 | `{程序名}_{状态}_{版本号}_{构建日期}_{构建序号}.{扩展名}` | `{程序名}_{版本号}_{状态}_{构建日期}_{构建序号}.{扩展名}` |
| 英文 | `{app_name}_{status}_{version}_{date}_{seq}.{ext}` | `{app_name}_{version}_{status}_{date}_{seq}.{ext}` |
| 示例 | `Jeenith_fix_2.10.3_20260722_01.apk` | `Jeenith_2.10.3_fix_20260722_01.apk` |

## 影响分析

### 影响范围

- **下次构建起**：build_apk.ps1 产出的 APK 文件名将采用新格式 `Jeenith_{版本号}_{状态}_{日期}_{序号}.apk`
- **archive_history.py**：能正确解析新格式 APK 文件名并追加到双份 build_history.json；不再解析旧格式（v2.10.3 及之前的产物如需补录历史需手动操作）
- **build_apk.ps1 Get-BuildSequence 正则**：扫描 builds/android/ 计算序号时只匹配新格式；旧格式 APK 不会被误匹配（已用 `Jeenith_fix_2.10.3_20260722_01.apk` 验证不匹配）

### 不影响

- **已发布的 GitHub Release**：release_history.json 中的 assets.name / downloadUrl 字段保留旧名，不破坏下载链接
- **已归档的 APK / Windows zip 文件**：builds/android/ 与 builds/windows/ 下的历史产物保留旧文件名，不重命名
- **build_history.json 历史条目**：filename / targetPath / filePath 等字段值保留旧名（真实历史记录）
- **release_notes/*.md**：已发布版本的发布说明描述的是历史事实，不改
