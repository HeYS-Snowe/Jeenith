# 批量重命名现有构建产物为新命名格式

**类型**：config
**日期**：2026-07-22 11:30
**影响范围**：builds/android/、builds/windows/、builds/build_history.json、mobile/build_history.json、builds/release_history.json、docs/（10 文件）、builds/release_notes/（36 文件）

## 变更描述

将所有现有构建产物从旧格式 `{程序名}_{状态}_{版本号}_{日期}_{序号}` 批量重命名为新格式 `{程序名}_{版本号}_{状态}_{日期}_{序号}`，并同步更新所有引用旧格式产物名的衍生内容。此操作承接 commit 7babb11 的命名规则字段互换变更。

## 修改详情

- [builds/android/]：重命名 37 个 APK 文件（status 和 version 字段互换）
- [builds/windows/]：重命名 36 个 zip 文件（status 和 version 字段互换）
- [builds/build_history.json]：更新 65 条 build 记录的 filename/filePath/targetPath/sourcePath 字段（118 处替换）
- [mobile/build_history.json]：同步更新 65 条 build 记录（98 处替换）
- [builds/release_history.json]：更新 71 个 assets 的 name/localPath 字段（142 处替换）；downloadUrl 保持旧格式（GitHub Release 未重传，71 处保留）
- [docs/]：更新 10 个文件的 123 处引用（PROJECT_SUMMARY 大表 60 处、TRAR_IDE 功能更新与bug修复 25 处、部署运维手册 12 处、运维手册 7 处、工作流程 5 处、Git 协作规范 5 处、性能测试报告 4 处、NEXT_PLAN 2 处、测试报告 2 处、知识库 1 处）
- [builds/release_notes/]：更新 36 个 release_notes 文件的 asset 清单表（71 处替换）

## 影响分析

- 构建功能不受影响（build_apk.ps1 的 Get-BuildSequence 用新格式正则扫描目录，能正确匹配重命名后的文件）
- release_history.json 中 downloadUrl（旧格式，GitHub 保留）与 localPath（新格式，本地改名）的文件名部分不一致，这是预期行为
- GitHub Release notes 框中的内容保持旧格式（不重传），与本地 release_notes 文件不一致，这是预期行为
- log/update/ 历史更新日志未回溯修改，保留历史事实记录原貌
