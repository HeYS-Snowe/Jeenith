# GoldButton / DarkButton 订阅 configProvider 使动画开关即时生效

**类型**：fix
**日期**：2026-07-14 20:05
**影响范围**：mobile/lib/shared/widgets/gold_button.dart、mobile/lib/shared/widgets/dark_button.dart

## 变更描述

修复 `GoldButton` 与 `DarkButton` 未订阅 `configProvider` 导致动画开关不即时生效的问题。原实现通过 `_animEnabled` getter 调用 `ref.read(configProvider)`：在 `build` 中读取当前值但**不订阅**，当用户在设置页切换 `animationsEnabled` 后，按钮不会重建，仍按旧开关状态运行（动画/静态形态不会切换），需等到下次外部触发重建才生效。

改为 `build` 中 `ref.watch(configProvider)` 订阅 + 实例字段缓存，与上一轮 `AnimatedExpandIcon` 修复模式一致。

## 修改详情

两个文件修改完全对称：

- **[gold_button.dart](file:///d:/Code/Project/Qore/Jeenith/mobile/lib/shared/widgets/gold_button.dart)** / **[dark_button.dart](file:///d:/Code/Project/Qore/Jeenith/mobile/lib/shared/widgets/dark_button.dart)**
  - 移除 getter `bool get _animEnabled => ref.read(configProvider)...`
  - 新增实例字段 `bool _animEnabled = true;`，并附注释说明用途（供 tap 回调安全读取，避免非 build 上下文调用 `ref.watch` 反模式）
  - `build` 顶部新增 `_animEnabled = ref.watch(configProvider).valueOrNull?.animationsEnabled ?? true;`，订阅配置变更并缓存
  - `_onTapDown` / `_onTapUp` / `_onTapCancel` 保持读 `_animEnabled` 字段不变（这些回调在非 build 上下文触发，必须读缓存而非 `ref.watch`）

## 设计要点

Issue 示例建议直接在 `build` 中 `final _animEnabled = ref.watch(...)`。但本组件的 `_animEnabled` 同时被三个 tap 回调（`_onTapDown` / `_onTapUp` / `_onTapCancel`）使用——这些回调在非 build 上下文执行，若 getter 用 `ref.watch` 会在 tap 时触发 Riverpod 反模式（在非 build 上下文建立订阅）。

故采用「build 中 `ref.watch` 订阅 + 实例字段缓存 + tap 回调读字段」模式：
- 设置页切换开关 → `configProvider` 通知 → `build` 重建 → 字段更新 → 下次 tap 用新值
- tap 回调永远读最近一次 build 的缓存值，不触碰 `ref.watch`

此模式与 [animated_expand_icon.dart](file:///d:/Code/Project/Qore/Jeenith/mobile/lib/shared/widgets/animated_expand_icon.dart)（fix_20260714_01.md）完全一致，三个动效组件现已统一。

## 影响分析

- **行为修复**：设置页切换 `animationsEnabled` 后，GoldButton/DarkButton 立即按新开关状态工作（动画形态 ↔ 静态形态无缝切换），无需重新进入页面
- **无 API 变更**：构造参数、公开行为不变，所有调用方无需修改
- **性能**：`ref.watch` 订阅成本极低，仅在 `configProvider` 变更时触发本组件一次重建
- **一致性**：项目三个动效组件（GoldButton / DarkButton / AnimatedExpandIcon）现已全部采用订阅模式
