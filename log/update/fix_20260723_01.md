# DarkButton 浅色适配补齐 + configProvider 订阅修复 + SvgIcon fallback 主题感知

**类型**：fix
**日期**：2026-07-23 11:30
**影响范围**：mobile/lib/shared/widgets/dark_button.dart、mobile/lib/shared/widgets/svg_icon.dart

## 变更描述

补齐 v2.10.0 浅色适配遗漏的两个共享组件，并修复 HANDOFF 任务 B（DarkButton 的 configProvider 订阅问题）：

1. **DarkButton 主题感知**：此前仍用 v2.0.0 旧色板（紫黑渐变 + 浅金字硬编码 `Color(0xFF...)`），浅色模式下与设置页/GoldButton 配色不一致，混用深浅元素。本次改为 `AppClr.of(context)` 主题感知：深色沿用紫黑渐变 + 浅金字，浅色切换为浅米渐变（`AppColorsLight.buttonTop/buttonBottom`）+ 深棕字（`0xFF2E2210`），与设置页配色基准对齐。

2. **DarkButton configProvider 订阅修复（HANDOFF 任务 B）**：`_animEnabled` 旧实现是 `bool get _animEnabled => ref.read(configProvider)...` getter，`ref.read` 不建立订阅，设置页切换 `animationsEnabled` 开关后按钮不会即时重建（动画/静态形态不切换）。改为 **build 中 `ref.watch` 订阅 + 实例字段缓存** 模式（与 GoldButton / AnimatedExpandIcon 统一），tap 回调读字段缓存以避免在非 build 上下文调用 `ref.watch`。

3. **SvgIcon fallback 主题感知**：fallback 色此前用 `theme.color` 硬编码，无 IconTheme 时无主题感知。改为 `context.appClr.textPrimary`，浅色模式下回退为深棕字（`0xFF2E2210`），对比度充足。

- **浅色模式视觉统一**：DarkButton 与 GoldButton / 设置页配色基准一致，不再混用深浅元素，符合用户偏好（完整浅色适配，不混用深浅元素，以设置页配色为基准）
