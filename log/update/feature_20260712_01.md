# 桌面端窗口尺寸自适应（像台手机立在屏幕上）

**类型**：feature
**日期**：2026-07-12 17:30
**影响范围**：mobile/lib/main.dart、mobile/windows/runner/main.cpp、mobile/pubspec.yaml

## 变更描述

桌面端（Windows/macOS/Linux）启动时按当前屏幕分辨率动态计算窗口尺寸：高度撑满屏幕（留少量边距给任务栏），宽度按手机竖屏比例 9:18.5 自适应，使 App 在桌面端呈现"手机竖屏"形态。允许用户拖拽调整大小，设最小尺寸 380×600 防止 UI 压崩。移动端 / Web 完全不受影响。

## 修改详情

- [mobile/pubspec.yaml](file:///d:/Code/Project/Qore/Jeenith/mobile/pubspec.yaml)：新增 `window_manager: ^0.5.2`（桌面端窗口管理）+ `screen_retriever: ^0.2.2`（获取屏幕分辨率）。
- [mobile/lib/main.dart](file:///d:/Code/Project/Qore/Jeenith/mobile/lib/main.dart)：在 `main()` 中判断 `PlatformInfo.isDesktop`，桌面端分支调用 `windowManager.ensureInitialized()` → `ScreenRetriever.instance.getPrimaryDisplay()` 拿屏幕物理尺寸 → 除以 `scaleFactor` 得逻辑像素 → 高度扣 40px 边距后 clamp 到 [600, 1400] → 宽度按 9:18.5 比例算 → `WindowOptions(size, minimumSize=380×600, center=true)` + `waitUntilReadyToShow` 后再 `show()` 避免闪烁。移动端 / Web 不走该分支，无影响。
- [mobile/windows/runner/main.cpp](file:///d:/Code/Project/Qore/Jeenith/mobile/windows/runner/main.cpp)：Windows 初始窗口尺寸从 `1280×720`（横屏）改为 `460×900`（竖屏），与 Dart 端目标值接近，避免启动时窗口从横屏闪烁 Resize 到竖屏。注释用英文避免 MSVC C4819 编码警告。

## 影响分析

- **桌面端用户体验**：启动后窗口呈手机竖屏形态，高度撑满屏幕，居中显示，可拖拽调整但不会缩到 380×600 以下。
- **移动端 / Web**：完全无影响——`PlatformInfo.isDesktop` 守护，且 window_manager / screen_retriever 在这些平台不调用任何 API。
- **构建**：Android 构建不受影响（依赖正常解析，Dart 端无平台调用）；Windows 构建已验证通过（`flutter build windows --debug` 成功生成 `jeenith.exe`）。
- **依赖体积**：新增 2 个桌面端专用包，APK 不增加原生代码体积（plugin 在 Android 上无原生实现），Windows zip 体积变化可忽略。
- **平台扩展性**：macOS / Linux 桌面端也自动适配（window_manager 跨平台），无需额外配置。
