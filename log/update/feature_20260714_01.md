## 变更描述

结束 1.x 的「功能堆叠」阶段，转向 2.x 的「交互体验与极致细节」阶段。本次为 major 升级，落地 NEXT_PLAN_20260713-230018_01.MD 规划的四个模块：核心微交互升级（按钮物理反馈 + 图标状态切换动画 + 设置页动效配置）、UI 资产检查、算法与内容资产稳定性审计、工程化构建与发行归档。

按用户约束实现：动画必须高级、不能显得廉价；可设置页全局开关；无光污染，配色遵循 oklch 色彩思路；选择自定义 AnimationController 而非引入 flutter_animate，与现有 InteractableCard 风格一致。

## 修改详情

### 模块一：核心微交互升级

#### 1.1 动效常量与工具封装
- [mobile/lib/core/theme/animations.dart]（新增）：集中管理所有微交互的时长 / 曲线 / 错峰间隔
  - `AppAnimations.pressDown = 110ms` / `pressRelease = 260ms`（按动抬起用 easeOutBack 弹回）
  - `iconRotate = 240ms` / `iconScale = 140ms`（图标状态切换）
  - `cardStagger = 90ms` / `cardRise = 420ms` / `panelExpand = 260ms`
  - `pressReleaseCurve = Cubic(0.34, 1.56, 0.64, 1)`（easeOutBack 变体，黑金低饱和度配色思路）
  - `staggeredIntervals(count, stepRatio, durationRatio)` 工具方法，end 用 `.clamp(0.0, 1.0)` 守卫，避免 Interval end > 1 崩溃

#### 1.2 按钮物理反馈重写
- [mobile/lib/shared/widgets/gold_button.dart]（重写）：从 StatelessWidget → ConsumerStatefulWidget，内部自动读 configProvider，**调用方零改动**
  - `SingleTickerProviderStateMixin` + `AnimationController` 提供 vsync
  - onTapDown：`_press.forward()` + setState 阴影变小
  - onTapUp：`_press.reverse().then(() => onPressed())`，先弹回再触发回调（避免按下视觉中断）
  - build：`AnimatedBuilder` + `Transform.scale(0.95 + 0.05 * upCurve.value)` + boxShadow 阴影变化
  - `_animEnabled` getter 读 configProvider，关闭时直接跳过动画（不影响功能）
- [mobile/lib/shared/widgets/dark_button.dart]（重写）：同款升级
  - 配色：紫黑渐变 `[Color(0xFF3A2F4A), Color(0xFF241C30)]` + 鎏金描边
  - 与 GoldButton 同结构，调用方零改动

#### 1.3 图标状态切换动画
- [mobile/lib/shared/widgets/animated_expand_icon.dart]（新增）：
  - `expanded` / `onPressed` / `color` / `size` / `expandedAngle`（默认 90）/ `animationsEnabled` / `collapsedIcon`（默认 `Icons.expand_more`）
  - 点击瞬间 `_breath.forward().then((_) => _breath.reverse())` 触发 `AnimatedScale` 呼吸反馈（缩放 1.0 → 0.86 → 1.0）
  - `AnimatedRotation(turns: expanded ? expandedAngle/360 : 0)` 平滑切换图标角度
  - 关闭动效时直接渲染静态 Icon，避免任何视觉跳变

#### 1.4 配置加动效开关
- [mobile/lib/core/config/app_config.dart]：新增 `animationsEnabled` 字段（默认 true），影响构造函数 + `defaults` 静态常量 + `copyWith`
- [mobile/lib/core/config/config_providers.dart]：
  - `build()` 读取 `animationsEnabled`（SharedPreferences key `'animationsEnabled'`，fallback true）
  - 新增 `setAnimationsEnabled(bool v)` 方法：写 prefs + 更新 state

#### 1.5 设置页动效总开关
- [mobile/lib/features/settings/settings_page.dart]：在「外观」和「动画设置」之间新增「◆ 动效」section
  - SwitchListTile：「启用微交互动效」，subtitle 说明「按钮按动缩放/图标状态切换/卡片错峰等微交互（关闭后不影响功能）」
  - 改 `SwitchListTile` 加 `shape: RoundedRectangleBorder(borderRadius: 14)` + Container `clipBehavior: Clip.antiAlias`，保持圆角点击区域一致
  - 设置页底部「保存配置」按钮由 `ElevatedButton` 改为 `GoldButton`，统一品牌质感

#### 1.6 ExpansionTile 替换为带动画的版本
- [mobile/lib/features/home/home_page.dart]：

## 影响分析

- **交互层面**：按钮按下 0.95 缩放 + 抬起 easeOutBack 弹回，图标点击带呼吸反馈 + 旋转切换，整体从「生硬」升级到「丝滑有质感」

- **性能**：AnimationController 配合 SingleTickerProviderStateMixin，每个按钮/图标独立 vsync，关闭动效时直接跳过 setState，无额外开销

## 相关问题

实现 `docs/NEXT_PLAN/NEXT_PLAN_20260713-230018_01.MD` 规划的 v2.0.0 四模块。

### 实施过程中的关键决策

1. **按钮动画与 config 联动方式**：最初设计给 GoldButton/DarkButton 加 `animationsEnabled` 参数，但发现 30+ 调用方都需要传参。改为 ConsumerStatefulWidget 内部自动读 configProvider，调用方零改动
2. **ExpansionTile 默认 chevron 与 trailing 冲突**：Flutter 的 ExpansionTile 默认 chevron 会覆盖 trailing 字段。解决方案：传 `trailing` 参数会覆盖默认 chevron。配合 `ExpansibleController` 外部控制 + `onExpansionChanged` 同步状态
3. **flutter_animate vs 自定义动画**：评估后选择自定义。理由：项目已有 InteractableCard 自定义动画风格，避免新依赖增加 APK 体积，与现有视觉一致性，性能可控
4. **ExpansionTileController 废弃**：Flutter 3.31+ 起 `ExpansionTileController` 已废弃，改用 `ExpansibleController`
5. **Windows zip 体积增长**：v1.8.0 13.06 MB → v2.0.0 21.00 MB，因 ConsumerStatefulWidget 引入 Riverpod 引用与 Flutter 版本变化。功能正常，接受此大小
