# v2.10.1 修正 v2.10.0 GoldButton/DarkButton 双层嵌套 BUG

**类型**：fix
**日期**：2026-07-21
**影响范围**：mobile/lib/shared/widgets/gold_button.dart、mobile/lib/shared/widgets/dark_button.dart、docs/频发BUG/GoldButton竖线坍塌.md

## 变更描述

v2.10.0 引入的 `ConstrainedBox(minWidth:88, maxWidth:inf) + SizedBox(width:inf)` 双层嵌套反向放大了竖线坍塌 BUG，导致周易页面所有按钮消失、小六壬 DarkButton 失去宽度约束。本次修正简化为 `ConstrainedBox(minWidth:88)` 单层结构。

## 修改详情

- [mobile/lib/shared/widgets/gold_button.dart]：expandedBox 从双层嵌套简化为 `ConstrainedBox(constraints: BoxConstraints(minWidth: 88), child: box)` 单层；更新文档注释说明 v2.10.1 修正历史与三场景验证
- [mobile/lib/shared/widgets/dark_button.dart]：expandedBox 从双层嵌套简化为 `ConstrainedBox(constraints: BoxConstraints(minWidth: 72), child: box)` 单层；更新文档注释
- [docs/频发BUG/GoldButton竖线坍塌.md]：追加 v2.10.0 错误方案与 v2.10.1 简化方案的"四次根治"历史；复用排查清单更新（明确不应有 maxWidth:inf / SizedBox(width:inf)）；预防措施追加"警惕加层思维"；版本演进时间线追加两行

## 影响分析

### 根因（v2.10.0 错误方案）
`ConstrainedBox + SizedBox(width:inf)` 在 loose(maxWidth=W) 约束场景下：
1. ConstrainedBox enforce 后传 child (minWidth:88, maxWidth:W)
2. SizedBox(width:inf) 收到后 enforce → minWidth=clamp(88, inf, inf)=inf, maxWidth=clamp(W, inf, inf)=inf → child 收 tight(inf)
3. box.size=(inf, h)，SizedBox.size=(inf, h)
4. ConstrainedBox.size=clamp(inf, 0, W)=W → 强制撑满整个 Row 剩余宽度
5. 结果：Row 后续按钮（DarkButton/CopyResultButton/ShareResultButton）无空间全部消失；DarkButton 在 Wrap 中撑满整行

### 修正方案（v2.10.1）
仅保留 `ConstrainedBox(minWidth:88)` 单层，去掉 `maxWidth:inf` 和 `SizedBox(width:inf)`。
- tight(W) 下：enforce → tight(W)，撑满 W（Expanded / SizedBox(width:88) 包裹场景）
- loose(maxWidth=W) 下：enforce → (minWidth:88, maxWidth:W)，box 按 intrinsic 测量，minWidth 兜底
- loose unbounded(maxWidth=inf) 下：enforce → (minWidth:88, maxWidth:inf)，box 按 intrinsic 测量，minWidth 兜底

### 验证
- flutter analyze：0 issue
- 周易 _buildActionBar：GoldButton('摇卦') 用 Expanded 包裹（tight 约束）→ 撑满 W；DarkButton/CopyResultButton/ShareResultButton 在 Row 中无 Expanded（loose 约束）→ 按 intrinsic 宽度
- 小六壬 _buildActionBar：GoldButton('起卦') 用 SizedBox(width:88) 包裹（tight(88) 约束）→ 撑满 88；DarkButton 在 Wrap 中（loose 约束）→ 保持 intrinsic 宽度
