# v2.10.2 修复 InteractableCard 崩溃 + DarkButton 文字过渡动画

**类型**：fix + feature
**日期**：2026-07-21
**影响范围**：mobile/lib/features/home/home_page.dart、mobile/lib/shared/widgets/dark_button.dart、mobile/pubspec.yaml

## 变更描述

修复首页 InteractableCard 入场动画 `Interval.begin` 越界导致的 `'begin <= 1.0'` 断言崩溃；为 DarkButton 内部 label 增加 AnimatedSwitcher 包裹，text 变化时自动播放淡入+上滑过渡动画（配合 CopyResultButton 图标切换）。

## 修改详情

- [mobile/lib/features/home/home_page.dart]：techs 卡片入场 `Interval` 的 `begin = 0.28 + i * 0.08` 增加 `.clamp(0.0, 0.92)`（原仅 end 有 clamp，begin 在 i>=10 时 >1.0 触发断言崩溃）
- [mobile/lib/shared/widgets/dark_button.dart]：label 用 `AnimatedSwitcher`（key: ValueKey(text)）包裹，text 变化时淡入+上滑过渡（250ms）；更新文档注释追加 v2.10.2 说明
- [mobile/pubspec.yaml]：版本号 2.10.0+40 → 2.10.2+41

## 影响分析

### 崩溃根因
首页 techs 列表用 `for (var i = 0; i < techs.length; i++)` 生成 InteractableCard，入场 `Interval(0.28 + i*0.08, ...)`。当前项目 12+ 卜算术，i 可达 11+，`begin = 0.28 + 11*0.08 = 1.16 > 1.0`。`Interval.transformInternal` 在 debug 模式断言 `begin <= 1.0` 失败抛出异常。偶发是因为仅 i>=10 的卡片越界。

### 崩溃修复
begin 增加 `.clamp(0.0, 0.92)`，与 end 的 `.clamp(0.0, 1.0)` 配合，确保：
- begin ∈ [0, 0.92]，end ∈ [0, 1.0]
- end - begin >= 0.08（避免除零）
- i 较大时多个卡片共享 [0.92, 1.0] 区间（仅影响入场时机，不崩溃）

### 文字过渡动画
DarkButton 内部 label 从直接 `Text(widget.text)` 改为 `AnimatedSwitcher(child: Text(key: ValueKey(widget.text)))`。
- text 不变时：AnimatedSwitcher 不触发动画，无副作用（如"重置"按钮）
- text 变化时：淡入+上滑过渡（250ms），与 CopyResultButton 图标旋转动画（400ms）呼应
- 不受 `AppConfig.animationsEnabled` 控制（与图标动画一致，属 UI 操作反馈）

### 验证
- flutter analyze：0 issue（22.3s）

## 相关问题

- 首页 InteractableCard 偶发崩溃（`'begin <= 1.0': is not true`）
- CopyResultButton 文字过渡动画需求（用户反馈"SVG 图标有过渡动画，给文字也加一个"）
