# 修复 Windows 桌面端窗口启动尺寸过小 + 首页卡片溢出

**类型**：fix
**日期**：2026-07-12 23:20
**影响范围**：mobile/lib/main.dart、mobile/windows/runner/main.cpp、mobile/lib/features/home/home_page.dart

## 变更描述

用户反馈 Windows 启动时窗口尺寸过小，并触发首页 `Column overflow by 51 pixels` 异常。
定位根因：`screen_retriever` 的 `Display.size` / `Display.visibleSize` 已经是逻辑像素
（DPI-aware），但 `main.dart` 误把它当物理像素又除以 `scaleFactor`，导致计算出的
目标窗口尺寸偏小（1080p 屏幕 dpr=1.25 时，目标高度被算成 651 而非应有的 1040）。
窗口过窄又触发首页 GridView 卡片可用空间不足（约 84.7×104），Column 内容溢出。

## 修改详情

- **main.dart** — [main.dart](file:///d:/Code/Project/Qore/Jeenith/mobile/lib/main.dart)
  - 移除错误的 `physical.height / dpr` 除法（screen_retriever 返回的就是逻辑像素）
  - `_kMinWindowSize` 从 `Size(570, 900)` 改为 `Size(420, 700)`
    原 570 比目标 targetWidth (~477) 还大，导致 minimumSize 把窗口拉宽到 570
    反而违反"像手机"的窄屏设计意图；现 420 让 targetWidth 能生效

- **main.cpp** — [main.cpp](file:///d:/Code/Project/Qore/Jeenith/mobile/windows/runner/main.cpp)
  - 初始尺寸 `460×900` → `540×1040` 物理像素
    更接近 main.dart 计算的目标尺寸，减少启动闪现"小→大"跳变

- **home_page.dart** — [home_page.dart](file:///d:/Code/Project/Qore/Jeenith/mobile/lib/features/home/home_page.dart#L225)
  - 卡片内容紧凑化（防御性修复，即使窄窗口也不溢出）：
    - 图标 46×46 → 40×40，Icon size 加 22 显式约束
    - displayName fontSize 26 → 22
    - subtitle fontSize 12 → 11
    - description maxLines 3 → 2，并用 `Flexible` 包裹让窄窗口下优雅降级
    - SizedBox(height: 8) → 6，height 1.4 → 1.35

## 影响分析

- **核心修复**：1080p 屏幕 dpr=1.25 下，窗口尺寸从错误的 ~299×651 修正为正确的 ~477×1040，符合"高度撑满 + 手机比例"的设计
- **首页稳定**：即使未来出现更窄窗口（如用户手动拖小），卡片也不会再 Column overflow
- **启动体验**：main.cpp 初始尺寸接近最终值，启动闪现更平滑
